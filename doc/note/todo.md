# TODOリスト

## 構成要素

### ユーザ
- ユーザID
- 名前

### プロジェクト
カテゴリ的な感じっぽい、TODOタスクの入れ物
- プロジェクトID
- プロジェクト名

### TODOタスク
- データ
    - タスクID
    - 件名 (必須)
    - 内容 (必須)
    - 状態 / (完了 or 未完了)
    - 日時の情報だけで判断できないか検討すること
    - 期日
    - 作成日時
    - 完了日時

- 機能
    - 件名変更
    - 内容変更
    - 状態変更
    - 期日追加、変更
    - 並び順変更
    - 優先度変更
    - 別プロジェクトに移動

- 拡張項目
    - 優先度
    - 並び順
